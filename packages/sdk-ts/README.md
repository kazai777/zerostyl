# @zerostyl/sdk-ts

TypeScript SDK for the [ZeroStyl](https://github.com/kazai777/zerostyl) zk toolkit on Arbitrum Stylus.

> **Status — v0.1.0 (alpha scaffold).** This release ships the codegen pipeline that turns a circuit's `AbiSchema` JSON into TypeScript types your dApp can import. The proving runtime (WASM bundle of the Rust prover) and on-chain submission helpers (ethers v6) land in subsequent releases, once the universal on-chain verifier is in place.

## What's in this release

- **Runtime types** mirroring the Rust `AbiSchema` (`Hex`, `FieldType`, `WitnessSchema`, `PublicInputsSchema`, `ProvingSystem`, …) — see [`src/types.ts`](src/types.ts).
- **`generateBindings(abi)`** — TypeScript codegen that turns an `AbiSchema` into a ready-to-commit `.ts` module with a `<Name>Circuit` const + `<Name>Witness` / `<Name>PublicInputs` interfaces.
- **`zerostyl-sdk generate` CLI** — wraps the codegen with `--abi` / `--out` flags.

## Install

> **Not yet on npm.** Publishing lands with a later release. For now, use the package from a clone of the ZeroStyl monorepo (see [Use from the monorepo](#use-from-the-monorepo) below). Once published, install will be the standard:
>
> ```bash
> pnpm add -D @zerostyl/sdk-ts   # planned, not live yet
> ```

Node ≥ 20 (the SDK is ESM-only).

## Use from the monorepo

Clone the repo, install workspace dependencies, build the SDK once, then invoke the CLI directly via Node:

```bash
git clone https://github.com/kazai777/zerostyl.git
cd zerostyl
pnpm install
pnpm --filter @zerostyl/sdk-ts build

# Generate bindings from an abi.json produced by `zerostyl-export transform`:
node packages/sdk-ts/bin/zerostyl-sdk.js \
  generate \
  --abi examples/zk_private_demo/abi.json \
  --out /tmp/circuit.ts
```

CLI flags:

| Flag | Short | Description |
|---|---|---|
| `--abi <path>` | `-a` | Path to the `abi.json` file (required) |
| `--out <path>` | `-o` | Output `.ts` file. If omitted, writes to stdout (pipe-friendly) |

## What the generator emits

For a circuit named `deposit` with one `#[zk_private(commit = "poseidon")]` parameter:

```typescript
// auto-generated by @zerostyl/sdk-ts — do not edit

import type { Hex } from '@zerostyl/sdk-ts';

export const DepositCircuit = {
  name: "deposit",
  version: "1.0.0",
  defaultK: 10,
  numPublicInputs: 1,
  numPrivateWitnesses: 3,
} as const;

export interface DepositWitness {
  collateral: bigint;
  collateral_nonce: Hex;
  threshold: bigint;
}

export interface DepositPublicInputs {
  collateral_commitment: Hex;
}
```

## Type mapping

`generateBindings` maps the Rust `FieldType` discriminated union to TypeScript as follows:

| `FieldType` | TypeScript | Notes |
|---|---|---|
| `u64`, `u128` | `bigint` | beyond `Number.MAX_SAFE_INTEGER` |
| `bool` | `boolean` | |
| `fp`, `bytes32`, `address` | `Hex` | `` `0x${string}` `` template literal type |
| `array<T, N>` | `ReadonlyArray<T>` | recursive; length lives in JSDoc / runtime metadata |

`Hex` is exported from `@zerostyl/sdk-ts` and gives compile-time validation that a string starts with `0x`.

## Programmatic API

Once the package is on npm, you can drive the codegen from a build script:

```typescript
import { readFileSync, writeFileSync } from 'node:fs';
import { generateBindings, parseAbiSchema } from '@zerostyl/sdk-ts';

const abi = parseAbiSchema(readFileSync('abi.json', 'utf8'));
writeFileSync('src/generated/circuit.ts', generateBindings(abi));
```

From a monorepo clone, replace the `@zerostyl/sdk-ts` import with the relative path to `packages/sdk-ts/src/index.ts` (or add the workspace package as a `workspace:*` dependency).

## Development

```bash
pnpm install
pnpm --filter @zerostyl/sdk-ts lint
pnpm --filter @zerostyl/sdk-ts build
pnpm --filter @zerostyl/sdk-ts test
```

Regenerate the committed snapshot after intentional codegen changes:

```bash
REGEN_SDK_TS_SNAPSHOTS=1 pnpm --filter @zerostyl/sdk-ts test
```
