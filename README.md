## zerostyl-debug

Inspect circuit structure and debug failing witnesses with readable diagnostics.

<!-- GIFs generated with VHS — see docs/assets/*.tape -->
![inspect demo](docs/assets/inspect.gif)

![debug demo](docs/assets/debug.gif)

Measured on real circuits: **−45% output lines · −74% noise lines · −61% struct nesting · −95% chars to gate name · gate name on the first failure line**.

```bash
# Reproduce the benchmark
cargo run -p debug-workflow-bench --release -- --verbose
```

```bash
# Inspect circuit structure (columns, gates, constraints, degree)
zerostyl-debug inspect --circuit state_mask

# Show expected witness JSON format for a circuit
zerostyl-debug schema --circuit state_mask

# Display witness values and derived public inputs
zerostyl-debug witness --circuit state_mask --witnesses witnesses/state_mask_valid.json

# Debug a failing witness — shows which gate fails and why
zerostyl-debug debug --circuit state_mask --witnesses witnesses/state_mask_wrong_commitment.json
```

Available circuits: `state_mask`, `tx_privacy`, `private_vote`

## zerostyl-prove

Generate and verify halo2 proofs off-chain. Uses the same witness files as `zerostyl-debug`.

![prove demo](docs/assets/prove.gif)

```bash
# Generate a proof
zerostyl-prove generate --circuit state_mask --witnesses witnesses/state_mask_valid.json

# Verify a proof (public_inputs.json auto-generated by generate)
zerostyl-prove verify --circuit state_mask --proof proof.bin

# Show circuit info and expected witness format
zerostyl-prove info state_mask
```

Available circuits: `state_mask`, `tx_privacy`, `private_vote`

Proving keys are cached in `.zerostyl_cache/` after the first run.

## Circuits

| Circuit | What it proves |
|---------|---------------|
| `state_mask` | A secret value lies within [min, max] without revealing it |
| `tx_privacy` | A token transfer is valid — balance conservation + Merkle membership |
| `private_vote` | A voter holds sufficient balance and cast a valid boolean vote |

### Witness files

```bash
# See the expected JSON format for any circuit
zerostyl-debug schema --circuit state_mask
```

Example (`witnesses/state_mask_valid.json`):
```json
{ "state_value": "42", "nonce": "123", "range_min": "0", "range_max": "255" }
```

Inject a deliberate failure with `_debug` overrides:
```json
{ "state_value": "42", "nonce": "123", "range_min": "0", "range_max": "255",
  "_debug": { "commitment": "999" } }
```

Seven example files covering valid and broken witnesses are in `witnesses/`.
